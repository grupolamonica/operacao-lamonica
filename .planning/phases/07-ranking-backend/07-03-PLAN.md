| Boundary | Description |
|----------|-------------|
| Torre backend → Supabase ride-rank | Reads via service_role (Plan 01). Dados retornados incluem PII de motoristas (driver_name) e avaliações. |
| Torre backend → docs.google.com (gviz) | Fetch de CSV PÚBLICO de terceiro (Google). Conteúdo é não-confiável (planilha editável por terceiros) e entra no parser. |
| Redis (cache) | CSV parseado fica em Redis (`ranking:sheets:trips`). Contém dados operacionais de viagens. |
