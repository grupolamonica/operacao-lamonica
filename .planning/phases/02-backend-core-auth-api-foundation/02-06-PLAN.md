**api/src/index.ts** — main composition file:
```typescript
import { Elysia } from 'elysia'
import { cors } from '@elysiajs/cors'
import { swagger } from '@elysiajs/swagger'

import { logger } from './lib/logger'
import { authPlugin } from './modules/auth/auth.plugin'
import { tripsPlugin } from './modules/trips/trips.plugin'
import { driversPlugin } from './modules/drivers/drivers.plugin'
import { alertsPlugin } from './modules/alerts/alerts.plugin'
import { vehiclesPlugin } from './modules/vehicles/vehicles.plugin'
import { dashboardPlugin } from './modules/dashboard/dashboard.plugin'

const PORT = Number(process.env.PORT ?? 3000)
const FRONTEND_URL = process.env.FRONTEND_URL ?? 'http://localhost:5173'

export const app = new Elysia()
  // ---- CORS — RESEARCH Pitfall #2: explicit origin required when credentials:true ----
  .use(cors({
    origin: FRONTEND_URL,                                        // exact string, not '*'
    credentials: true,
    methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
    allowedHeaders: ['Content-Type', 'Authorization'],
  }))
  // ---- Swagger / OpenAPI ----
  .use(swagger({
    path: '/swagger',
    documentation: {
      info: {
        title: 'Torre de Controle API',
        version: '0.2.0',
        description: 'API de monitoramento de entregas em tempo real',
      },
      tags: [
        { name: 'auth',      description: 'Autenticação (HttpOnly Cookie JWT)' },
        { name: 'trips',     description: 'Viagens e KPIs' },
        { name: 'drivers',   description: 'Motoristas e KPIs' },
        { name: 'alerts',    description: 'Alertas + tratativas' },
        { name: 'vehicles',  description: 'Frota' },
        { name: 'dashboard', description: 'KPIs agregados (Redis cache)' },
      ],
    },
  }))
  // ---- Health ----
  .get('/', () => ({ status: 'ok', service: 'torre-api', version: '0.2.0' }))
  // ---- Error handler ----
  .onError(({ code, error, set }) => {
    if (set.status === 401 || set.status === 403 || set.status === 404 || set.status === 429) {
      return { error: error.message }
    }
    logger.error({ code, error: error.message }, 'unhandled error')
    set.status = 500
    return { error: 'Internal server error' }
  })
  // ---- Modules (composed BEFORE type App is captured — RESEARCH Pitfall #5) ----
  .use(authPlugin)
  .use(tripsPlugin)
  .use(driversPlugin)
  .use(alertsPlugin)
  .use(vehiclesPlugin)
  .use(dashboardPlugin)
  .listen(PORT, () => {
    logger.info({ port: PORT, frontendUrl: FRONTEND_URL }, 'torre-api listening')
  })

// CRITICAL — Eden Treaty type export (D-04). Must be after all .use() calls.
export type App = typeof app
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → / (health) | Public, no auth, returns minimal info — no service version leak risk |
| browser → /swagger | Public, lists routes/schemas — accept (internal app, dev-only is wrong; doc) |
| browser → /api/* | Protected by authGuard inside each plugin |
| frontend (5173) → api (3000) | CORS-controlled with credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06-01 | Information Disclosure | Public Swagger UI exposes API structure | accept (MVP) | Internal app; no PII in docs. Production hardening in Phase 6 will gate /swagger behind auth or restrict by env. Documented. |
| T-02-06-02 | Tampering | CORS misconfiguration accepts cross-origin requests with cookie | mitigate | `origin: FRONTEND_URL` is exact-match string. Production must override `FRONTEND_URL` to the prod domain — documented in .env.example. |
| T-02-06-03 | Denial of Service | Unbounded request body size | accept | Elysia default body limit (1MB) is sufficient for current endpoints. Will revisit in Phase 6 when telemetry POST is added. |
| T-02-06-04 | Information Disclosure | Stack traces leaked via 500 responses | mitigate | `onError` handler returns `{error: 'Internal server error'}` for status 500; logger captures full error server-side only. |
| T-02-06-05 | Tampering | Eden Treaty type export creates monorepo coupling — frontend could import runtime accidentally | mitigate | `torre-de-controle/src/types/api.ts` uses `export type { App }` (type-only). Phase 3 must use `import type { App }`. Anti-pattern documented in 02-RESEARCH.md (Eden Treaty client-only). |
| T-02-06-06 | Repudiation | Login attempts not audit-logged | accept | logger.info captures success/failure with email + ip. Persistent audit log deferred to Phase 6. |
