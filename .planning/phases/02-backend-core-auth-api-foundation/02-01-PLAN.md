**api/Dockerfile**:
```dockerfile
FROM oven/bun:1.4

WORKDIR /app

# Install deps first (cache layer)
COPY package.json bun.lockb* ./
RUN bun install --frozen-lockfile || bun install

# Copy source
COPY . .

EXPOSE 3000

CMD ["bun", "run", "src/index.ts"]
```

**docker-compose.yml** at the worktree root:
```yaml
services:
  postgres:
    image: postgres:16-alpine
    container_name: torre-postgres
    environment:
      POSTGRES_DB: torre_controle
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres -d torre_controle"]
      interval: 5s
      timeout: 3s
      retries: 10

  redis:
    image: redis:7-alpine
    container_name: torre-redis
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 10

  api:
    build:
      context: ./api
      dockerfile: Dockerfile
    container_name: torre-api
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgres://postgres:postgres@postgres:5432/torre_controle
      REDIS_URL: redis://redis:6379
      JWT_SECRET: ${JWT_SECRET:-dev-secret-change-in-production-please-make-this-very-long-and-random-64-bytes-hex}
      JWT_EXPIRES_IN: 24h
      NODE_ENV: development
      PORT: 3000
      FRONTEND_URL: http://localhost:5173
      LOG_LEVEL: debug
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    volumes:
      - ./api:/app
      - /app/node_modules

volumes:
  pgdata:
```

- API CORS origin already wired to `FRONTEND_URL` (RESEARCH Pitfall #2: explicit origin required with credentials:true).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host shell → docker daemon | Local dev only; no exposure to internet |
| docker network → external | Only ports 3000/5432/6379 exposed to host (dev) |
| .env files → repository | `.env` in .gitignore; only `.env.example` checked in |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Information Disclosure | Hard-coded secrets in repo | mitigate | `.env` git-ignored; `.env.example` carries placeholders only; `JWT_SECRET` injected at runtime via shell env. Documented in .env.example that production must replace placeholder. |
| T-02-01-02 | Tampering | Compose default `JWT_SECRET` reused in production | mitigate | Default value contains "change-in-production" string; production deploy must override. Documented in .env.example. |
| T-02-01-03 | Denial of Service | Unbounded postgres/redis port binding | accept | Local dev only; ports bound to localhost on host. Production deploy will use Railway/Render private networking. |
| T-02-01-04 | Elevation of Privilege | Bun container running as root | accept | Dev-only image; production hardening (USER directive, non-root) deferred to Phase 6 deploy plan. |
</threat_model>
